3 # Throwing smart pointers
4 
5 A header only library wrapping [std::shared_ptr](http://en.cppreference.com/w/cpp/memory/shared_ptr) and [std::unique_ptr](http://en.cppreference.com/w/cpp/memory/unique_ptr) that throws an exception when a wrapped null pointer is dereferenced
6 
7 ## Rationale
8 
9 Dereferencing a null standard smart pointer is undefined behaviour, probably leading to a program crash.
10 This library aims to fully replicate the behaviour of the wrapped standard smart pointers with the exception (sic) that an exception will be thrown when the ->, * and [] operators are called. Such exceptions can be caught and handled in an application specific manner (for example by showing an error pop up)
11 
12 Situations where this library may be useful:
13 - Legacy code that used a custom smart pointer class which threw on dereference: this library provides a gradual path towards moving to the standard smart pointers. Such code may depend on the throwing behaviour, making a direct migration bug prone.
14 - Programs where resilience to null dereference bugs is important, such as GUIs
15 
16 There is an obvious cost to using this library in that every dereference will add a branch. In these cases the library exposes the underlying standard smart pointer and the user may opt to use that in hot code paths where attention has been taken not to dereference null pointers.
17 
18 ## Design decisions
19 
20 The library relies on the underlying implementation of std::shared_ptr and std::unique_ptr for all functionality, including participation in overload resolution. While this exposes users to slight differences in behaviour across platforms and compilers, the upside is that it's easier to give access to the stored std pointers and when the user decides to no longer rely on these classes and move to the std implementations, they will get less surprising results.
21 
22 Exceptions are throwing::make_unique and throwing::unique_ptr::operator<< which are provided by the implementation.

24 ## Using the library
25 
26 ### Accessing the functionality
27 
28 throwing_ptr is a header-only library. You can copy the include throwing_ptr folder in your source code or add the include directory in your include path and start using it right away.
29 
30 ### Examples
31 
32 Example use 1:
33 ```c++
34 #include <throwing/shared_ptr.hpp>
35 auto pointer = throwing::make_shared<int>();
36 ```
37 
38 Example use 2:
39 ```c++
40 #include <throwing/unique_ptr.hpp>
41 auto pointer = throwing::make_unique<SomeClass>(some_parameter);
42 ```
43 
44 Example use 3 (exception handling):
45 ```c++
46 #include <throwing/unique_ptr.hpp>
47 
48 // Initialise a null pointer
49 throwing::unique_ptr<int> null_int_ptr;
50 auto non_null_float_ptr = throwing::make_unique<float>(0);
51 try {
52  *non_null_float_ptr;
53  (*null_int_ptr)++; // dereference of null pointer, throws exception
54 } catch (const throwing::null_ptr_exception<float> &) {
55  // It is possible to handle exceptions differently based on the
56  // type of object handled by the pointer
57  // This block will not be called as it only catches
58  // dereferences of int pointers
59 } catch (const throwing::base_null_ptr_exception &e) {
60  // what() returns a generic message
61  std::cout << e.what() << std::endl;
62  // what_type() returns platform-specific message
63  // indicating the type
64  std::cout << e.what_type() << std::endl;
65 }
66 
67 ```
